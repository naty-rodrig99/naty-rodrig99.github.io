Sentiment Classification

Comparing Naive Bayesian and SVM algorithms in sentiment classification

OVERVIEW

This project compares and evaluates the performance of Naive Bayesian and SVM algorithms in sentiment classification. By analyzing user-generated textual data, specifically from Yelp reviews, aiming to enhance understanding of their methodologies and improve the extraction of user insights from textual data.

OBJECTIVE

Comparing Naive Bayesian and SVM algorithms in sentiment classification to reveal their unique approaches and enhance user insights.

PROBLEM STATEMENT

"How do Naive Bayesian and SVM algorithms compare in their effectiveness for sentiment classification, and what impact does this have on accurately extracting user insights from textual data?"

ITEM

Graduate project

METHODS

Naive Bayesian Classifier and SVM (Support Vector Machine) Classifier

TOOLS

Python

TEAM

Individual

TIME FRAME

2 weeks

METHODOLOGY

Algorithms

  • Naive Bayesian: Chosen for its simplicity and effectiveness in handling discrete frequency counts.
  • SVM (Support Vector Machine): Selected for its ability to classify linearly separable data.

Dataset

Utilized Yelp reviews as the primary source of textual data and was divided in:

  • Train: 7500 reviews
  • Evaluation: 5000 reviews
  • Test: 2500 reviews

Preprocessing

Implemented the following to clean the data:

  • Removed HMTL, tags, @, #, ‘http’, ‘www’.
  • Converted to lower case.
  • Removed punctuation and non-alphabetic characters.
  • Splitted into individual words.
  • Joined into a cleaned string.

Feature Extraction

TF-IDF (Term Frequency-Inverse Document Frequency):

  • This method was used to convert text into a matrix format, representing the importance of each word in the context of the documents.

IMPLEMENTATION

Naive Bayes Classifier

1. Training the Naive Bayes Classifier
  • Splits the training data into positive and negative sentiment classes based on the target labels.
  • Creates word frequency counts for each class, capturing how often each word appears in both positive and negative texts.
  • Class priors are calculated, representing the proportion of positive and negative samples in the dataset.
2. Predicting Sentiment
  • Used the trained model to calculate log probabilities for each class (positive and negative), ensuring numerical stability.
  • For each word in the text, it computes the likelihood of the word belonging to the positive or negative class, using frequency counts.
  • The class with the higher log probability is returned as the prediction (either positive or negative).
3. Model Evaluation
  • Applied the trained Naive Bayes model to the evaluation dataset, which contains 5,000 Yelp reviews from a different category than the training data. This tested the model's generalization ability.
  • The same process was repeated for the test dataset, and accuracy metrics were calculated for both datasets.

SVM (Support Vector Machine) Classifier

1. Label Preparation

Before training the SVM, the sentiment scores are converted into two classes:

  • Positive sentiment is labeled as 1.
  • Negative sentiment is labeled as -1.
2. SVM Model Training

Trained the linear SVM model using a stochastic gradient descent approach:

  • It initializes the weights and bias to zero.
  • For each iteration, the function updates the weights and bias based on whether a sample is correctly classified, adjusting them to maximize the margin between the positive and negative classes.
  • The learning rate and regularization parameters control the step size and penalty for large weights, respectively, improving the model's generalization.
3. Prediction
  • Computed the dot product between the feature vectors and the learned weights to classify each sample as positive or negative, outputs predictions as either 1 (positive) or -1 (negative).
4. Evaluation
  • Applying the trained SVM model to the evaluation dataset, which contains 5,000 Yelp reviews from a different category than the training data.
  • The same process was repeated for the test dataset, and accuracy metrics were calculated for both datasets.
